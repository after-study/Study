## 1-1 클로드 코드와 에이전틱 코딩

### 에이전틱 코딩
에이전틱 코딩은 AI가 시스템 탐색, 코드 수정, 테스트 실행까지 자율로 수행하는 접근 방식
탐색 - 계획 - 구현 - 커밋 워크플로

### 에이전틱 루프
클로드 코드는 에이전틱 루프라는 순환 구조로 작동한다.
컨텍스트 수집, 작업 수행, 결과 확인의 세 단계를 순환.

1. 컨텍스트 수집: 파일 구조, 코드, 설정 파일 탐색. **CLAUDE.md**에 기록된 프로젝트 규칙, 코드 스타일, 테스트 절차 등도 컨텍스트에 포함
2. 작업 수행: 
	1. 파일조작(Read, Write, Edit, MultiEdit)
	2. 검색(Glob, Grep)
	3. 셸 실행(Bash)
	4. 웹 접근(WebFetch, WebSearch)
3. 결과확인: 테스트 실행. 린터로  품질 점검. 빌드 완료 검사.
	1. 규칙 기반 피드백
	2. 시각적 피드백
	3. LLM 기반 평가

TDD 워크플로에서 이 반복구조가 명확히 드러난다.

### CLI 인터페이스의 차별점
```bash
$ cat security_logs_*.log | claude -p "이 로그들을 분석해서 해킹 시도 패턴을 찾아서 위협 수준과 함께 보고해줘."
```
1. 보안 로그 파일의 내용을 모두 읽고 출력
2. | (파이프): 앞 명령어 출력을 다음 명령어의 입력으로 전달
3. claude -p : 클로드 코드를 프롬프트와 함께 비대화형 모드로 실행
